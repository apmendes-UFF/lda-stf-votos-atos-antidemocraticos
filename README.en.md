# Topic Modeling (LDA) of the Rapporteur's Opinions — Criminal Cases Related to the January 8th Antidemocratic Acts

*[Leia isto em português](README.md)*

Reproducibility repository for the article *"Topic Modeling as a Tool for Analyzing the Content of Judicial Decisions: A Study of the Rapporteur's Opinions in Criminal Cases Related to the Antidemocratic Acts of January 8th"*.

Contains the R scripts used for data collection, preprocessing, LDA topic modeling, and analysis of Justice Alexandre de Moraes's opinions in 149 Brazilian Supreme Federal Court (STF) rulings related to the antidemocratic acts of January 8th, 2023.

## Repository structure

```
scripts/
  0_libraries.R                    # Required R packages
  1_opinionextraction.R            # Extraction of the rapporteur's opinion from the full rulings (PDF)
  2_pdftostring.R                  # Conversion of the extracted PDFs into plain text
  3_preprocessing.R                # Tokenization, stopword removal, and vocabulary standardization
  4_exploratorytextanalysis.R      # Exploratory analysis (word frequency, tf-idf)
  5_topicmodeling.R                # LDA model fitting, k selection, evaluation metrics, and word clouds
  6_metadataanalysis.R             # Cross-analysis of ruling metadata (sentence type, sex, page count, dominant topic)
data/
  sw_PT.xlsx                       # Portuguese stopword list
  stopwords_8janSTF.xlsx           # Custom stopword list built for this study
  metricsbyk.xlsx                  # Model fit metrics (perplexity, coherence, exclusivity) by k
  termotopic_Betamatrix.xlsx       # Beta matrix (term-topic probabilities)
  topicdoc_Thetamatrix.xlsx        # Dominant topic per document
  votos_tokens.xlsx                # Preprocessed token base (output of script 3)
figures/
  Figura1_paginas_x_pena.png              # Page count of opinions/rulings x sentence type
  Figura2_top20_palavras.png              # Top-20 most frequent words
  Figura3_top10_tfidf.png                 # Top-10 words by tf-idf
  Figura4_coerencia_exclusividade.png     # Coherence x exclusivity scatterplot (k selection)
  Figura5_nuvens_palavras_topicos.jpg     # Word clouds for the 6 topics (grid)
  Figura6_distribuicao_documentos_topicos.png  # Distribution of documents by topic
  Figura7_scatterplot_tempos_pena.png     # Sentence length by topic
  Figura8_dispersao_topicos_paginas.png   # Dispersion of topics by page count
```

**Note on personal data:** the files `metadata.xlsx` (which contains defendants' full names in the `reu` column) and `sw_deffendantsnames.xlsx` (name list used in the stopword removal step) **are not published in this repository**, as they contain identifiable personal data of defendants in criminal proceedings. They are available upon direct request to the authors.

## How to reproduce the analysis

The scripts should be run in numeric order (`0` through `6`). Each one assumes the output files of the previous script are present in the working directory:

1. **`0_libraries.R`** — installs the R packages used throughout the pipeline.
2. **`1_opinionextraction.R`** — starting from the full PDF rulings (not included in this repository, as they are large public files — they can be obtained directly from the [STF case law search page](https://jurisprudencia.stf.jus.br)), extracts only the pages corresponding to the rapporteur's opinion.
3. **`2_pdftostring.R`** — converts the extracted opinion PDFs into plain text.
4. **`3_preprocessing.R`** — tokenizes the text, removes stopwords (using the lists in `data/`), and applies vocabulary standardization (manual lemmatization of ~1,292 inflectional variants, resulting from an expert review of the vocabulary). Produces `votos_tokens.xlsx`.
5. **`4_exploratorytextanalysis.R`** — generates word frequency and tf-idf statistics (Figures 2 and 3 of the article).
6. **`5_topicmodeling.R`** — builds the document-term matrix, tests k from 2 to 14 topics (NPMI coherence, perplexity, and exclusivity), fits the final LDA model (k=6, α=0.16, η=1×10⁻⁴, Gibbs sampling, seed=123), and generates the per-topic word clouds (Figure 5).
7. **`6_metadataanalysis.R`** — cross-references each document's dominant topic with the ruling metadata (sentence type, defendant's sex, page count).

## Data

The original PDF rulings are not redistributed in this repository, as they are large files directly and publicly available on the STF website. The intermediate matrices (tokens, beta, theta) and the fit metrics are available in `data/`, allowing the topic modeling and analyses to be reproduced from the preprocessing stage onward without reprocessing the original PDFs. The complete metadata table (including defendant identification) can be made available upon direct request to the authors, in order to safeguard third parties' personal data.

## Figures

The `figures/` folder contains the 8 figures generated by the scripts and used in the article (Figures 1 through 8), numbered to match the final version of the text, allowing direct comparison between the repository and the published article.
